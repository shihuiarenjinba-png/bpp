## 基本概念

### キャッシュフロー（CF）の分類

企業のキャッシュフローは3つの活動に分類されます：

#### 1. 営業CF（Operating Cash Flow, OCF）
- **定義**: コア事業活動から生まれるキャッシュ
- **計算**: 純利益 + 減価償却費 + 調整項目 - 運転資本増加分
- **意味**: 企業の本業がどれだけキャッシュを生み出しているかを示す
- **健全性**: 継続的に正の営業CFが理想

#### 2. 投資CF（Investing Cash Flow, ICF）
- **定義**: 資産の購入・売却によるキャッシュフロー
- **計算**: ー固定資産投資 + 資産売却益
- **意味**: 企業の成長投資の規模を示す
- **通常**: マイナス（成長企業は積極的に投資）

#### 3. 財務CF（Financing Cash Flow, FCF）
- **定義**: 融資、株式発行、配当金支払いなどのキャッシュ
- **計算**: 借入金増減 + 株式発行 - 配当金支払い
- **意味**: 資本調達と還元の活動を示す
- **柔軟性**: 経営判断で調整可能

### 合計CF計算式

```
合計CF（月）= 営業CF + 投資CF + 財務CF
```

## データ分析手法

### 1. 移動平均（Moving Average）

**目的**: 短期的な変動ノイズを除去し、真のトレンドを抽出

**計算式（3ヶ月移動平均）**:

```
MA₃(t) = (CF(t) + CF(t-1) + CF(t-2)) / 3
```

**解釈**:
- 単純移動平均：等しい重みで過去データを平均
- 直近3ヶ月の「平均的な」キャッシュフロー状況を把握
- 月次変動が激しい場合は有効

**活用例**:
```
5月の移動平均 = (5月CF + 4月CF + 3月CF) / 3
        = (500万 + 400万 + 600万) / 3
        = 500万円
```

### 2. トレンド分析（Trend Analysis）

**目的**: 長期的な方向性（上昇/下降/横ばい）を把握

**計算方法（最小二乗法による線形回帰）**:

```
CF(t) = a + b×t + ε

ここで：
  a = 切片（初期値）
  b = 傾き（月次トレンド）
  t = 経過月数
  ε = 誤差項
```

**傾き(b)の計算**:

```
b = Σ[(t - t̄)(CF - CF̄)] / Σ[(t - t̄)²]

t̄  = 平均月数
CF̄ = 平均キャッシュフロー
```

**解釈**:
- b > 0: CFが月平均で b 円ずつ改善（ポジティブトレンド）
- b < 0: CFが月平均で |b| 円ずつ悪化（ネガティブトレンド）
- |b| が大きい: トレンドが強い

**活用例**:
```
過去12ヶ月のデータ分析結果：
  傾き b = 250,000円/月

解釈: 月平均で25万円ずつキャッシュフローが改善している
→ 現在のペースが続けば12ヶ月後には+300万円の改善
```

### 3. 季節分解（Seasonal Decomposition）

**目的**: CFの周期的パターンを特定し、予測精度を向上

**構成要素**:

```
CF(t) = Trend(t) × Seasonal(t) + Noise(t)

Trend     : 長期傾向
Seasonal  : 月次パターン（1年周期）
Noise     : ランダム変動
```

**季節性指数（Seasonal Index）の計算**:

```
季節性指数₁₂月 = 12月の平均CF / 全月の平均CF

例：
  12月の平均 = 800万円
  年間平均   = 500万円
  季節指数   = 800 / 500 = 1.6

→ 12月は年間平均の160%、つまり20%高い
```

**月別パターン認識**:

| 月 | 季節性指数 | パターン | 業種例 |
|:--:|:----------:|:--------:|--------|
| 1月 | 0.7 | 売上低迷 | 小売・飲食 |
| 4月 | 0.9 | やや低迷 | 多くの業種 |
| 7月 | 1.3 | ピーク前 | 製造業 |
| 12月 | 1.5 | 最ピーク | 小売・電子商取引 |

## 予測モデル

### 基本予測フレームワーク

本スクリプトの予測は、季節性とトレンドを組み合わせた加算モデルを使用：

```
予測CF(t) = 直近平均CF × 季節性指数 + トレンド調整
```

### 1. 直近平均CFの選定

```
基準CF = (過去3ヶ月の営業CF + 過去3ヶ月の投資CF + 過去3ヶ月の財務CF) / 3
```

**理由**:
- 3ヶ月: 短期すぎず（ノイズを含む）、長期すぎず（古い情報）のバランス
- 最新の経営状況を反映しつつ、1ヶ月の異常値の影響を軽減

### 2. 季節性指数の適用

```
季節調整CF = 基準CF × 季節性指数(月)
```

**例**:
```
基本CF (3ヶ月平均) = 600万円
7月の季節性指数 = 1.2

7月の季節調整CF = 600万 × 1.2 = 720万円
```

### 3. トレンド調整の追加

```
最終予測CF = 季節調整CF + (トレンド傾き × 予測月数)
```

**例**:
```
基本予測       = 720万円
トレンド傾き   = 25万円/月
予測6ヶ月目    = 720万 + (25万 × 6) = 870万円
```

## シナリオ分析

### 3シナリオの定義

本スクリプトは以下の3シナリオで予測を提供：

### 1. 基本シナリオ（Base Case）

**定義**: 現在の傾向が継続すると仮定

```
基本シナリオ = 計算された予測CF（上記参照）
```

**使途**: 
- 最も可能性が高いシナリオ
- 経営計画の立案基準
- 予算編成のベース

**仮定**:
- 市場環境が大きく変わらない
- 現在の経営施策が継続される
- 季節変動は過去と同程度

### 2. 楽観シナリオ（Optimistic Case）

**定義**: 良好な経営環境を仮定（基本の+15%）

```
楽観シナリオ = 基本シナリオ × 1.15
```

**適用場面**:
- 売上拡大施策が成功した場合
- 新製品が想定以上に売れた場合
- コスト削減施策が効果を発揮した場合
- 市場環境が改善した場合

**シナリオ例**:
```
基本予測   = 700万円
楽観予測   = 700万 × 1.15 = 805万円

差分 = +105万円
```

### 3. 悲観シナリオ（Pessimistic Case）

**定義**: 経営が一時的に悪化（基本の85%）

```
悲観シナリオ = 基本シナリオ × 0.85
```

**適用場面**:
- 主要顧客の喪失
- 予期しない原価上昇
- 市場需要の低下
- 人員流出による効率低下

**シナリオ例**:
```
基本予測   = 700万円
悲観予測   = 700万 × 0.85 = 595万円

差分 = −105万円
```

### シナリオの活用

予測期間の現金ポジションを3シナリオで計算：

```
累積CF = Σ月次CF（各シナリオ）

終了時現金 = 期首現金 + 累積CF
```

**経営意思決定**:
- 基本シナリオが正常に推移する現金計画
- 楽観シナリオが達成可能か検討
- 悲観シナリオでも耐えられる融資枠を確保

## リスク指標

### 1. キャッシュランウェイ（Cash Runway）

**定義**: 現在の現金で何ヶ月間事業を継続できるか

**計算式**:

```
キャッシュランウェイ = 現在の現金残高 / 月平均バーンレート

単位: ヶ月
```

**解釈基準**:

| ランウェイ | 判定 | 対応 |
|:----------:|:----:|------|
| 12ヶ月以上 | 健全 | 通常計画継続 |
| 6～12ヶ月 | 注意 | 短期利益計画強化 |
| 3～6ヶ月 | 警告 | 融資申込検討 |
| 3ヶ月未満 | 危機 | 即座の資金調達必須 |

**計算例**:
```
現在の現金       = 10,000万円
月平均バーン     = 200万円/月
ランウェイ       = 10,000 / 200 = 50ヶ月

→ 約4年間運営可能（但し赤字継続下で）
```

### 2. バーンレート（Burn Rate）

**定義**: キャッシュが消費される月平均速度（赤字企業向け指標）

**計算式**:

```
バーンレート = Σ直近N月の負CF / N

N = 通常3ヶ月
負CFのみを集計（営業CFが負の場合）
```

**例（赤字企業）**:
```
過去3ヶ月の営業CF: −500万、−400万、−600万

バーンレート = (500万 + 400万 + 600万) / 3
             = 500万円/月

→ 現金10,000万円だと20ヶ月で枯渇
```

**戦略的対応**:
- バーンレート削減施策（コスト削減）
- 売上増加施策（歳入増）
- 外部融資による現金補充

### 3. 現金ポジション（Cash Position）

**定義**: 各時点での累積キャッシュ残高

**計算式**:

```
現金ポジション(t) = 初期現金 + Σ(CF月次合計)

累積CF(t) = Σ₁ᵗ CFᵢ
```

**追跡方法**:

```
月   | CF      | 累積CF   | 現金ポジション
-----|---------|----------|---------------
初期 | —      | —       | 10,000万円
1月 | +200万  | +200万  | 10,200万円
2月 | −100万  | +100万  | 10,100万円
3月 | −300万  | −200万  | 9,800万円
```

**重要な監視点**:
- 現金ポジション が危機水準（例：初期の20%以下）に近づくと警告
- 複数シナリオで悲観シナリオが負になる可能性

## ワーキングキャピタル管理

### 定義

**ワーキングキャピタル（WC）** = 流動資産 - 流動負債

企業が短期的に事業を回すために必要な現金

### ワーキングキャピタル・サイクル（WC Cycle）

```
WCサイクル = 売上債権回転日 + 在庫回転日 - 買入債務回転日
```

**各要素**:

| 項目 | 計算式 | 意味 |
|------|--------|------|
| 売上債権 | 売掛金 × 365 / 売上 | 客から代金を回収するまでの日数 |
| 在庫 | 在庫 × 365 / COGS | 商品を在庫として抱える日数 |
| 買入債務 | 買掛金 × 365 / COGS | 仕入先への支払い猶予日数 |

**資金繰りへの影響**:

```
例：小売業
  売上債権回転日 : 3日（ほぼ現金販売）
  在庫回転日     : 30日
  買入債務回転日 : 45日

WCサイクル = 3 + 30 - 45 = −12日

→ 支払いより代金回収が12日先
→ キャッシュポジティブ（望ましい状態）
```

```
例：建設業
  売上債権回転日 : 90日（請求-支払いまで）
  在庫回転日     : 0日（受注生産）
  買入債務回転日 : 30日

WCサイクル = 90 + 0 - 30 = 60日

→ 60日間のつなぎ融資が必要
→ この間のキャッシュフローを予測で把握
```

### キャッシュコンバージョンサイクル（CCC）

**定義**: 現金支出から回収まで何日かかるか

```
CCC = WCサイクル

日数が短い = 資金効率が良い
日数が長い = つなぎ融資が必要
```

**改善施策**:

| 施策 | 効果 |
|------|------|
| 売上債権回転日短縮 | 代金回収の迅速化（早期割引など） |
| 在庫回転日短縮 | 在庫最適化（在庫管理システム導入） |
| 買入債務回転日延長 | 支払条件交渉（仕入先との関係構築） |

## 計算例：総合シミュレーション

### サンプルデータ

```
過去12ヶ月平均（3ヶ月最新）:
  営業CF : 600万円/月
  投資CF : −100万円/月
  財務CF : 0万円/月
  合計CF : 500万円/月

傾き : 20万円/月（改善トレンド）

現在の現金 : 3,000万円

7月の季節性指数 : 1.2
```

### 7月の予測計算

```
【基本シナリオ】
  基準CF = (600 − 100 + 0) = 500万円
  季節調整 = 500万 × 1.2 = 600万円
  トレンド調整 = 20万円 × 6ヶ月 = 120万円
  基本予測 = 600万 + 120万 = 720万円

【楽観シナリオ】
  楽観予測 = 720万 × 1.15 = 828万円

【悲観シナリオ】
  悲観予測 = 720万 × 0.85 = 612万円
```

### 現金ポジション予測（6ヶ月後）

```
初期現金 = 3,000万円

6ヶ月累積CF:
  基本 = 700万 + 710万 + 720万 + 730万 + 740万 + 750万
       = 4,350万円
  楽観 = 4,350万 × 1.15 = 5,003万円
  悲観 = 4,350万 × 0.85 = 3,698万円

6ヶ月後の現金:
  基本 = 3,000万 + 4,350万 = 7,350万円 ✓
  楽観 = 3,000万 + 5,003万 = 8,003万円 ✓✓
  悲観 = 3,000万 + 3,698万 = 6,698万円 ✓

→ 全シナリオで現金ショートなし、健全状態
```
