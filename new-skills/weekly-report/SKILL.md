# 週次レポート

## スキルの目的
このスキルは、ユーザーから提供された1週間の活動メモ（文字起こしや箇条書き形式）を基に、上司に提出可能な形式の週次活動報告書を自動で作成することを目的としています。出力は、指定されたWordテンプレートのレイアウトを忠実に再現します。

## 入力
ユーザーは、1週間の活動内容をまとめたテキストデータを提供します。これは、会議の議事録、日々の作業ログ、箇条書きのメモなど、どのような形式でも構いません。

## 出力フォーマット
出力は、`/mnt/skills/user/weekly-report/template_weekly_report.docx` にあるWordテンプレートのレイアウトを再現します。以下にその主要な構造を示します。

```
# 週次活動報告書

| 項目 | 内容 |
|------|------|
| 報告者 | {報告者名} |
| 報告日 | {YYYY/MM/DD} |

## 概要
{今週の活動を3行程度で要約}

## 主な活動内容
| 曜日 | 活動内容 |
|------|----------|
| 月 | {月曜日の活動} |
| 火 | {火曜日の活動} |
| 水 | {水曜日の活動} |
| 木 | {木曜日の活動} |
| 金 | {金曜日の活動} |

## 課題・所感
{課題や気づき、改善提案を記載}

## 来週の予定
- {予定1}
- {予定2}
- ...

---
本文: {文字数}文字 / 作成日: {YYYY/MM/DD}
```

## 出力ルール
- 報告書全体は**ですます調**で統一されます。
- 提供された活動メモが整理されていない文字起こしや箇条書きであっても、内容を適切に解釈し、報告書として読みやすく、かつ論理的に整理された文章を生成します。
- 各セクション（概要、主な活動内容、課題・所感、来週の予定）は、提供された情報に基づいて具体的に記述されます。
- 「報告者名」はユーザーからの指示がない場合、会話の文脈やuserMemoriesから名前を取得する。取得できない場合は報告者欄を空白のままにし、ユーザーに確認する。
- 「報告日」および「作成日」は、報告書作成時の日付が自動で挿入されます。
- 「文字数」は、報告書本文（「---」より上の部分）の総文字数が自動でカウントされ、記載されます。
- **Wordテンプレートのレイアウト（タイトル、報告者テーブル、各セクション見出し、フッターの文字数表記）を忠実に再現し、プレースホルダ部分を実際の内容に置き換えます。**
- **Word文書生成後、`/mnt/skills/user/weekly-report/update_wordcount.py` スクリプトを使用して、報告書本文の正確な文字数をカウントし、フッターの「本文: XXX文字」の「XXX」部分にその数値を自動で埋め込みます。**

## 使用例
ユーザーは以下のようにスキルを呼び出すことができます。

```
週次レポートを作成してください。活動メモは以下の通りです。
[ここに活動メモを記載]
```

または、

```
weekly-reportスキルを使って、このメモから週次報告書を作成して。
[ここに活動メモを記載]
```

## 補足
このスキルは、ユーザーの週次報告書作成の手間を軽減し、効率的な情報共有を支援します。
