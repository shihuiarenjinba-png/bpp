# Japan Stock Analyzer スキル

## 概要

証券コードを入力としてIRバンクから財務データを収集し、バフェット流の財務分析を行うHTMLツールを生成するスキル。

> **他スキルとのトリガー判断**
> - 「**証券コードや会社名**が明示されている」→ このスキルを使用（自動でIRバンクから取得）
> - 「**CSVやExcelファイルがアップロードされている**」→ `corporate-finance-analysis` を使用
> - 「**DCF・ポートフォリオ・配当再投資**」が目的 → `securities-analysis` を使用
>
> このスキルで取得したCSVデータは、`corporate-finance-analysis` や `securities-analysis` の入力ファイルとしてそのまま流用できます（Step 3のCSVフォーマットが共通仕様）。

**5種類の出力パターン：**
1. `full` — 全指標を1画面に集約した統合分析レポート（デフォルト）
2. `profitability` — 収益性指標のみ（ROE/ROA/営業利益率/純利益率/EBITDA）
3. `safety` — 財務健全性のみ（自己資本比率/D/E比率/有利子負債推移）
4. `valuation` — バリュエーションのみ（PBR/PER/PSR/配当利回り/配当性向）
5. `cashflow` — キャッシュフロー分析のみ（営業CF/FCF/設備投資/CF品質）

## 実行手順

### Step 1: 証券コードとパターンを確認

ユーザーのメッセージから以下を読み取る：
- **証券コード**（4桁数字）: 指定なければ確認する
- **分析パターン**（未指定なら `full`）
- **年数**（未指定なら直近5期）

### Step 2: IRバンクからデータを収集

以下の4ページを順番にweb_fetchで取得し、百万円単位の数値を抽出する。

```
https://irbank.net/E{8桁EDINET}/results  # 業績・BS概要
https://irbank.net/E{8桁EDINET}/bs       # BS詳細
https://irbank.net/E{8桁EDINET}/cf       # CF詳細
https://irbank.net/E{8桁EDINET}/dividend # 配当・EPS・BPS
```

**EDINETコードの解決方針（正確性優先）：**

キャッシュは「速度ヒント」に過ぎず、**常に `https://irbank.net/{証券コード}` への実アクセスで正確なEDINETコードを取得・検証する**。

理由: EDINETコードは企業再編・上場廃止・コード変更により変わる可能性があるため、キャッシュのみに依存すると誤ったページを参照するリスクがある。IRバンクはFSA（金融庁）のEDINETデータをリアルタイムで反映するため、実取得が唯一の正確性担保となる。

**標準手順（全銘柄共通）：**
```
1. web_fetch("https://irbank.net/{証券コード}") を実行する
2. レスポンスのURLまたはページ内リンクから Eで始まる8文字のEDINETコードを抽出する
   （例: /E02144/ というパスが含まれる）
3. 抽出したEDINETコードを使って4ページを取得する
```

**速度ヒントキャッシュ（Step 1の補助として参照可、必ず手順2で検証すること）：**

> **キャッシュと実取得が不一致の場合**: 実取得値を正とする。上場廃止・経営統合・EDINET再付番が発生した可能性がある。

### Step 3: データをCSV形式に整形

取得したHTMLから数値を抽出し、以下のCSV形式に変換する。

```csv

営業利益,2197748,2995697,2725025,5352934,4796000

総資産,62267140,67688771,74303180,90114296,93601350
株主資本,23404547,26245969,28338706,34220991,35924826
有利子負債,25659635,26496358,29380273,36561780,38792879
営業CF,2727162,3722615,2955076,4206373,5218700

```

**単位注意事項：**
- 百万円単位の値: そのまま記載
- 「兆」単位の表記 → ×1,000,000 で百万円に変換（例: 45.1兆 → 45100000）
- 「億」単位の表記 → ×100 で百万円に変換（例: 4796億 → 479600）
- EPS・BPS・株価・配当金: 円単位のまま（分割調整済み）

### Step 4: 適切なHTMLツールを生成

パターンに応じてアセットファイルを読み込んでHTMLを出力する。

```
assets/full.html         → 統合分析レポート
assets/profitability.html → 収益性分析
assets/safety.html       → 財務健全性分析
assets/valuation.html    → バリュエーション分析
assets/cashflow.html     → キャッシュフロー分析
```

各HTMLにはCSVデータを `window.injectData(csvText, fmtHint)` で注入して自動分析を起動する。

### Step 5: 出力

1. 取得したCSVデータをコードブロックで表示（ユーザーが確認・コピペ用）
2. HTMLツールをアーティファクトまたはファイルとして提供
3. 主要指標のサマリーをテキストで補足（直近期のROE・PBR・FCF等）

## 計算式（全パターン共通）

| 指標 | 計算式 | 備考 |
|------|--------|------|
| 営業利益率 | 営業利益 ÷ 売上高 | |
| 純利益率 | 純利益 ÷ 売上高 | |
| EBITDA | 営業利益 + 減価償却費 | 減価償却がない場合は営業利益のみ |
| ROE | 純利益 ÷ 期首期末平均自己資本 | 前期データなければ期末EQで代用 |
| ROA | 純利益 ÷ 総資産 | |
| 自己資本比率 | 自己資本 ÷ 総資産 | |
| D/E比率 | 有利子負債 ÷ 自己資本 | |
| FCF | 営業CF − 設備投資（絶対値） | |
| FCF品質 | FCF ÷ 純利益 | 0.7以上で良好 |
| PBR | 株価 ÷ BPS | IRバンクは株式分割調整済み |
| PER | 株価 ÷ EPS | 0＜PER＜500 の範囲のみ有効 |
| 配当利回り | DPS ÷ 株価 | |
| 配当性向 | DPS ÷ EPS | |
| 売上成長率 | (今期−前期) ÷ |前期| | |

## 会計基準の判定

IRバンクのページに「IFRS」「米国基準」「日本基準」の記載があるので確認する。
- 「IFRS」または「（連結）IFRS」→ `IFRS_C`
- 「米国基準」→ `US_C`
- 「日本基準（連結）」→ `JP_C`
- 「日本基準（単体）」→ `JP_S`

---

## エラー処理

- EDINETコードが不明 → `https://irbank.net/{証券コード}` を取得してURLから特定
- データが取得できない期 → nullで埋める（グラフは自動スキップ）
- 単位が曖昧 → 前後の文脈と桁数から判断（売上高が数百〜数千億なら億単位）
- 株式分割がある期 → IRバンクは分割調整済みEPS/BPS/DPSを提供するのでそのまま使用
